USB Printer Service — Integration Guide

How your application talks to the printer service — over HTTP or the command line — to print receipts and open the cash drawer.

Overview

The service takes a JSON payload, renders it to a receipt image, and sends it to a USB ESC/POS printer. It offers two integration surfaces that share one rendering pipeline:

HTTP API

Send a JSON payload to POST /print. Best for POS or web integrations.

Command line

Invoke printer --payload from any process. Best for local scripts.

Both produce an identical receipt. Use HTTP when your app is remote or browser-based; use the CLI when it runs on the same machine as the printer.

Running the service

Start the HTTP server so other apps can reach it:

python main.py
# or, from the CLI, optionally with host:port
printer --serve
printer --serve localhost:6000

Host, port and debug come from the service settings (default 0.0.0.0:5000). CORS is enabled, so browser apps can call it directly.

Quick start

Print your first receipt with a single request:

curl -X POST http://localhost:5000/print ^
  -H "Content-Type: application/json" ^
  -d "{\"items\":[{\"name\":\"Gasohol 95\",\"amount\":38.25,\"quantity\":10}],\"transaction_info\":{\"received\":500}}"

A 200 response with the computed total confirms the job was sent to the printer.

{ "status": "printed", "total": 382.5 }

Endpoints

EndpointDescription
GET /healthLiveness probe. Returns { "status": "ok" }.
POST /printRender and print a receipt. The request body is the payload.
POST /open-drawerOpen the cash drawer. Returns { "status": "drawer opened" }.
GET /docsThis documentation page.

Print a receipt

Send a POST to /print with Content-Type: application/json and the payload (see Schema) as the body.

Example — Python

import requests

payload = {
    "header_info": {"Customer Name": "ACME", "Transaction": "TXN-1234"},
    "items": [{"name": "Gasohol 95", "amount": 38.25, "quantity": 10}],
    "transaction_info": {"received": 500},
}
r = requests.post("http://localhost:5000/print", json=payload, timeout=10)
r.raise_for_status()
print(r.json())   # {"status": "printed", "total": 382.5}

Example — JavaScript

const res = await fetch("http://localhost:5000/print", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    items: [{ name: "Gasohol 95", amount: 38.25, quantity: 10 }],
    transaction_info: { received: 500 },
  }),
});
const data = await res.json();   // { status: "printed", total: 382.5 }

Open the cash drawer

curl -X POST http://localhost:5000/open-drawer

Errors & status

On success you get 200. On failure the body is { "error": "..." }.

StatusMeaning
200Receipt rendered and sent to the printer.
400Invalid JSON, or the payload failed validation (e.g. missing items).
500Printer / hardware failure while printing.

Set a request timeout and retry on 500 — the printer may be momentarily unavailable.

Payload schema

The same JSON body is accepted by both POST /print and printer --payload.

{
    "rfid": "",
    "info-title": "Tax Invoice (ABB)",
    "header_info": {
        "Customer Name": "ACME",
        "Transaction": "TXN-1234"
    },
    "items": [
        { "name": "Gasohol 95", "amount": 38.25, "quantity": 10.0 }
    ],
    "footer_info": { "Points": "30" },
    "transaction_info": {
        "received": 500.00,
        "change": 127.50,
        "discount": 10.00,
        "total": 382.50
    },
    "images": {
        "header": { "src": "data:image/png;base64,iVBORw0KGgo...", "scale": 100 },
        "footer": { "src": "assets/images/footer.png", "scale": 60 }
    }
}
FieldTypeNotes
rfidstringOptional. Printed at the top-left in the small font. Empty by default.
info-titlestringOptional. Printed centered after the header description in the main font size.
header_infoobjectOptional key/value pairs above the items. Recognized keys auto-translate (see Localization).
itemsarray Required, non-empty. Each needs name, amount (unit price) and quantity. Line total = amount × quantity.
footer_infoobjectOptional key/value pairs below the total. Recognized keys auto-translate.
transaction_infoobjectOptional. Only received, change, discount, total are read (see Auto-calculation).
imagesobjectOptional per-print header/footer images (see Images).

Legacy payload format

If any of customer, transection, promotion, points, or extras is present at the top level, the payload is auto-converted to the structure above.

Auto-calculation

When transaction_info is partial, missing values are derived from the items total:

  • If total is omitted → sum of amount × quantity across items.
  • If discount is given and total is omitted → total = items_total − discount.
  • If received and total are known but not change → change = received − total.
  • If change and total are known but not received → received = total + change.
  • If both received and change are given → total = received − change (authoritative).
  • All derived monetary values are rounded to 2 decimals.

Additionally, the resolved received / change / discount are appended to the footer, and a Value (pre-VAT) and VAT 7% line appear when the total is positive.

Images

Supply per-request header/footer images via the images field. Each slot accepts:

  • src — a file path or a base64 string / data: URI.
  • scale — render size as a percent of width (0–100).

Precedence via the API: payload images override the saved layout config. (When printing via the CLI, the --header-image / --footer-image flags additionally override the payload.)

Localization

The API renders receipts in the server's configured locale (Layout → receipt_locale: en or th). The CLI can override per-print with --locale en|th. Recognized keys in header_info / footer_info map to localized labels (case-insensitive); unknown keys print verbatim.

CLI integration

Instead of HTTP, a local app can invoke printer directly, passing the payload as a JSON string or a file path.

# From a payload file, on 80 mm paper, in Thai
printer --payload receipts/sale.json --paper-width 80 --locale th

# Inline JSON
printer --payload "{\"items\":[{\"name\":\"Gasohol 95\",\"amount\":38.25,\"quantity\":10}]}"

# Open the cash drawer
open-drawer

Exit codes: 0 success, 1 runtime error (print / drawer / server), 2 usage or validation error (bad payload, invalid --port / --serve / --paper-width). Success messages start with [OK]; errors print [ERROR] to stderr.
